# User Manual: Status of Load (SOC) Monitoring Platform

## 1. Functional Description
The monitoring platform is a web-based dashboard designed to manage and monitor critical site infrastructure. It provides alarm correlation and, based on this correlation, allows access to camera images and indicators that display power status, battery status, and air conditioning system values.

-## 2. Dashboard Navigation and Main Modules
### Home / Alarms
- **Two boxes**: Displays events generated by alarm correlation. For example, if an open door alarm and a motion event are detected simultaneously, a consolidated event is generated. This behavior applies to all alarm combinations on the site. From a confirmed event, it is possible to directly access the corresponding camera using a view button.

- **Anomalies**: Displays recurring alarms generated by one or more sensors, without requiring correlation with other events. These generally correspond to alarms that are triggered repeatedly over a period of time. It includes the option to export data to an Excel file for analysis.

### Cameras
- **Camera Module**: Provides an interface for viewing access and transport cameras.

- ### Batteries
- **Battery**: Displays the charge status (below 50%) of all batteries.

- **Power Failure**: Displays batteries experiencing a power outage. This information is shown in an easy-to-understand table, indicating the region and battery charge status. Additional site information can also be viewed in a pop-up window.

- **Display Battery**: Displays battery capacity information for all regions.

- **Voltage**: Displays rectifier voltage information for all regions.

- ### HVAC
- **Air Conditioning**: Displays information about the status of air conditioners, including temperature and humidity. It displays a very intuitive table that allows you to view the region and status of the air conditioners.

- ### System
- **System Status**: Displays the system status.

- **Edit Camera**: Provides CRUD (Create, Read, Update, Delete) functionality for camera information.

- **Configuration**: Provides CRUD functionality for users, including privilege assignment.

- ## 3. Common User Flows
### Case A: Alarm Monitoring
1. Log in to the control panel.

2. View completed events, whether access or transport events.

3. Identify the site with personnel present.

### Case B: Monitoring Sites with AC Outages
1. Log in to the control panel.

2. Filter the AC outage table by **Region** or **Type**.

3. Identify the site with the lowest **Voltage** or **Battery Percentage**.

4. Right-click or use the action button to copy the site data for a support ticket.

## 4. Frequently Asked Questions (FAQ)
- **Q: Why do some cameras appear offline but are visible in PEIM?**
- A: This can occur because the IP address information is incomplete or due to different credentials.

- **Q: Why do some batteries show "NO DATA"?**
- A: This occurs when the site rectifier or monitoring device (FSU/PECM) is disconnected or when the SNMP connection is blocked.

- **Q: Can I change the device settings through the control panel?**
- A: No. This platform is read-only for telemetry; configuration must be done through the device's native interface.
